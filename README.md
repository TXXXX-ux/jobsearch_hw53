В рамках этой части проекта была полностью переписана логика регистрации пользователей. Теперь система не просто записывает данные, но и следит за их уникальностью.

Что я тут сделал? Итак:

В UserRepositor добавил поддержку кастомных запросов. Теперь мы умеем проверять существование email в базе existsByEmail и искать юзера для авторизации findByEmail.
В UserService была внедрена бизнес-логика. Перед сохранением нового пользователя сервис проверяет, не занята ли почта. Если занята — выбрасывается RuntimeException, предотвращая дубликаты.
Тут я обновил UserController. Все эндпоинты таких как /users, /users/search, /users/register теперь связаны с сервисом и возвращают актуальные данные.
Чтобы сервер не ругался красными логами в консоль, добавлен глобальный обработчик ошибок. Теперь при попытке повторной регистрации клиент получает ответ 400 Bad Request с текстом Email уже занят!.

Как запустить:

Клонируем и открываем в IDEA.

Делаем Maven Reload ну на всякий пожарный, чтобы подтянулись зависимости.

И запускаем сервер через MovieRiviewerApplication

Чтобы не гадать, работает код или нет, я всё прогнал через Postman и Swagger. Теперь API не просто принимает запросы, а реально фильтрует то, что летит в базу.
